This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Contractor performance dashboard for LP First Capital. Scrapes stats from three portals (ContractorConnection, Sedgwick, Alacrity) using Playwright, stores data in an encrypted JSON file, and serves a password-protected static dashboard on GitHub Pages.

## Key Commands

```bash
# Daily update pipeline (decrypt → scrape all sources → re-encrypt data)
npm run update

# Update individual sources
npm run update:cc
npm run update:sedgwick
npm run update:alacrity

# Initial full scrape (all years 2020-2025, then encrypt everything including HTML)
npm run update:initial

# Encrypt/decrypt data manually
npm run decrypt          # stats.json.enc → stats.json
npm run encrypt:data     # stats.json → stats.json.enc (data only)
npm run encrypt          # Full encryption: data + StatiCrypt HTML wrapper

# Local dev server
npm run serve
```

All commands require `STATICRYPT_PASSWORD` in `.env`. Scraper commands also need portal credentials (`CC_USERNAME`, `CC_PASSWORD`, `CC_2FA_SECRET`, `SEDGWICK_*`, `ALACRITY_*`).

## Architecture

### Data Flow
1. **Scrapers** (`scraper/`) use Playwright to log into contractor portals and extract metrics
2. Data is saved to `data/stats.json` (plaintext, temporary)
3. **Encryption** (`scripts/encrypt-data.js`) encrypts to `data/stats.json.enc` using AES-CBC with PBKDF2 (600k iterations), then deletes plaintext
4. **Dashboard** (`index.html.bak`) decrypts data client-side using the password hash stored by StatiCrypt
5. GitHub Actions (`update-data.yml`) runs the pipeline daily at 6 AM UTC and commits `stats.json.enc`

### File Roles
- `index.html` - Encrypted (StatiCrypt-wrapped) dashboard, generated by `scripts/encrypt.js`. **Do not edit directly.**
- `index.html.bak` - The actual dashboard source code (HTML + inline JS). **Edit this file** for any dashboard/UI changes.
- `data/stats.json.enc` - Encrypted data file, the committed source of truth
- `data/stats.json` - Decrypted data, exists only temporarily during scrape runs
- `.staticrypt.json` - Salt used for encryption/decryption (must not change after initial setup)

### Scraper Structure
- `scraper/index.js` - Main orchestrator; runs CC, Sedgwick, and Alacrity scrapers
- `scraper/auth.js` - Auth0 login with TOTP 2FA for ContractorConnection
- `scraper/utils.js` - Shared utilities (data load/save, date formatting, diff calculation, `runStandalone()` helper)
- `scraper/cc.js`, `sedgwick.js`, `alacrity.js` - Standalone entry points for individual scrapers
- `scraper/tabs/` - CC tab-specific extraction logic (assignments, avgTip, poms, reinspections, surveys, qaFeedback)

### Dashboard (index.html.bak)
- Single-page app with all JS inline, uses Chart.js 4.4.1 and Tailwind CSS (CDN)
- Three data source views: ContractorConnection (default), Sedgwick, Alacrity
- CC view has time-period tabs (Daily/Weekly/Monthly/Yearly) with charts and tables
- `getSnapshotsByPeriod()` groups `dailySnapshots` into weekly/monthly aggregations
- `calculateSummary()` aggregates `byType` data into totals for charts

### Data Schema
```
statsData = {
  lastUpdated: ISO string,
  years: { "2026": { assignments: {byType: {...}}, poms: {byType: {...}}, ... } },
  dailySnapshots: [{ date: "YYYY-MM-DD", year, data, summary, diff, previousDate }],
  sedgwick: { dailySnapshots: [...] },
  alacrity: { dailySnapshots: [...] }
}
```

CC data is organized by **tab** (assignments, avgtip, poms, reinspections, surveys, qafeedback) and within each tab by **assignment type** (7 POM types like "Residential POMS", "Commercial POMS", etc.).

## Important Patterns

- Dates are stored as `YYYY-MM-DD` strings in Central Time (`America/Chicago`)
- After editing `index.html.bak`, run `npm run encrypt` to regenerate the encrypted `index.html`
- The encryption scripts share crypto logic (PBKDF2 rounds, AES-CBC) that must stay in sync across `encrypt.js`, `encrypt-data.js`, `decrypt.js`, and the injected client-side code
- `HEADLESS=false` env var runs scrapers with visible browser for debugging
